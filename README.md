# Условный подсчёт частоты слов (Word Frequency Task)

Высокопроизводительное C++ решение для многопоточного подсчета частоты слов в текстовом файле с применением криптографического хэширования (SHA-256).

## Архитектура и принятые решения

Решение построено на типичных для высоконагруженных систем приёмах: разделение ответственности, event-driven оркестрация и минимизации копирования данных.

### 1. Декомпозиция (SOLID / ООП)
Код разделен на независимые модули (классы) для соблюдения принципа единственной ответственности (SRP):
- **`main.cpp`** — чистая точка входа.
- **`WordFreqApp`** — фасад приложения (Singleton). Управляет жизненным циклом системных ресурсов, потоками и оркестрацией логики.
- **`ChunkProcessor`** — чистый вычислитель (Stateless Worker). Ничего не знает о мьютексах, файлах и главном потоке. Принимает кусок памяти (чанк), парсит его, вычисляет хэши и возвращает частотный словарь.
- **`Signal`** — обособленный инфраструктурный класс для реализации паттерна Observer.
- **`Constants.hpp`** — единая точка управления магическими константами (размеры буферов, имена файлов по умолчанию).
- **`sha256.hpp`** — компактная, header-only реализация SHA-256 (на базе кода Brad Conte), адаптированная для C++.

### 2. Паттерн Event Bus (Глобальная шина событий) и Event Loop
Для устранения проблемы "Head-of-Line Blocking" (когда быстрые потоки вынуждены ждать завершения самого медленного потока, если `std::future::get()` вызывается по порядку), применен паттерн **Event Bus** (Глобальная шина событий):
- Инфраструктура событий вынесена в независимый `namespace AppEvents` (`GlobalSignals.hpp`).
- Главный поток (`WordFreqApp`) при запуске подписывается на событие `AppEvents::OnChunkCompleted.Connect(...)`.
- По завершении работы воркер просто обращается к глобальной шине: `AppEvents::OnChunkCompleted.Emit(index)`. Воркер абсолютно изолирован (Loose Coupling) и не имеет ссылок на экземпляры или методы главного класса.
- Лямбда (выполняясь в ресурсах воркера) безопасно помещает индекс воркера в очередь (`m_readyIndices`) и будит главный поток через `std::condition_variable`.
- Главный поток реализует **Event Loop**: он спит, просыпается от сигналов воркеров, мгновенно забирает всю очередь (через `std::move`, чтобы не держать `std::mutex` и не блокировать другие воркеры) и в своем собственном темпе мержит данные.
Это дает максимальное перекрытие фаз Map (вычисления) и Reduce (слияние) и исключает простой ресурсов.

### 3. Максимальная производительность (High Performance / Zero-Copy)
- **Memory-Mapped Files (mmap / CreateFileMapping):** Файл не читается в промежуточные буферы через `std::ifstream`. Он проецируется напрямую в виртуальную память. Это позволяет программе молниеносно обрабатывать файлы размером в десятки гигабайт с нулевым оверхедом на копирование.
- **Zero-Copy парсинг:** Строки не выделяются в куче (heap allocation) при чтении. Слова накапливаются в стековых буферах (`char buffer[256]`) и напрямую скармливаются в математику `sha256_update`. Аллокация `std::string` происходит *исключительно* для слов, прошедших проверку хэшем, при их вставке в словарь.
- **Кастомный `IsAlpha` и `ToLower`:** Отказ от стандартных `std::isalpha` и `std::tolower` позволил:
  1. Избежать Undefined Behavior при работе с отрицательными значениями `char` (кириллица).
  2. Избавиться от накладных расходов на проверку системной C-локали (Locale) при посимвольном парсинге. Проверки инлайнятся компилятором в 2-3 ассемблерные инструкции.
- **Математика SHA-256:** Проверка условия «хэш заканчивается на символ 0..7 в hex-представлении» реализована через прямую битовую маску последнего байта (`(hash[31] & 0x0F) <= 7`). Это избавляет от тяжелой конвертации 32-байтного массива в 64-символьную Hex-строку.

### 4. Соглашения и допущения
- **Кодировка текста:** Реализована и оптимизирована поддержка однобайтовой кодировки **Windows-1251** (базовая латиница + кириллица А-Я, а-я, Ё, ё). Для поддержки UTF-8 потребовалось бы внедрение библиотеки ICU либо усложнение логики конечного автомата, что избыточно для рамок тестового задания.
- **Размер слова:** Длина одного слова ограничена `Config::MaxWordLength` (по умолчанию 255 символов) для возможности использования сверхбыстрых стековых буферов вместо динамической памяти. Если попадается непрерывная строка длиннее, она обрезается.
- **Количество потоков:** По умолчанию используется количество логических ядер процессора (`std::thread::hardware_concurrency()`), так как для математически емких задач (хэширование) это дает максимальный прирост скорости (Hyper-Threading).

## Подготовка, сборка и запуск

Проект использует CMake для кроссплатформенной конфигурации сборки.

### 1. Подготовка входного файла (ОБЯЗАТЕЛЬНО)

Перед первым запуском необходимо сгенерировать входной текстовый файл `input.txt` в **корне репозитория** (рядом с `CMakeLists.txt`). В репозитории нет готового файла из-за ограничений на размер. 
Для генерации файлов нужного размера в кодировке Windows-1251 (которая ожидается программой по условиям задачи) предусмотрен скрипт на Python:

```bash
git clone https://github.com/Alexuslo/word-freq.git
cd word-freq

# Сгенерирует файл input.txt размером 100 МБ
python generate_test_data.py

# Генерация файла специфичного размера (в мегабайтах) с кастомным именем
python generate_test_data.py --size 500 --output huge_test.txt
```

### 2. Сборка (Генерация проекта для Visual Studio)
Если вы хотите открыть код в Visual Studio, перейдите в корень клонированного репозитория и выполните следующие команды в консоли (потребуется установленный CMake):

```bash
mkdir build
cd build
cmake -G "Visual Studio 17 2022" -A x64 ..
```
После этого в папке `build` появится файл `WordFreqTask.sln`, который можно открыть в Visual Studio. Все заголовочные файлы `.hpp` также будут доступны в "Обозревателе решений" (Solution Explorer) для удобного чтения.

### Сборка через консоль
```bash
mkdir build
cd build
cmake ..
cmake --build . --config Release
```

### Запуск
```bash
# С явным указанием файлов:
.\Release\word_freq.exe <путь_к_исходному_файлу> <путь_к_результирующему_файлу>

# Использование файлов по умолчанию (input.txt -> result.txt):
.\Release\word_freq.exe
```
